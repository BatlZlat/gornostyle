# Проблема с merchantId

## Ситуация

Техподдержка дала `merchantId = MB0002168266` (13 символов), но API требует минимум 15 символов.

**Ошибка:**
```
"Field merchantId : String should have at least 15 characters; "
```

## Возможные причины

1. **MerchantId для API отличается от QR-кода СБП**
   - QR-код для СБП: `MB0002168266` (13 символов)
   - MerchantId для API интернет-эквайринга: должен быть 15 символов

2. **MerchantId опционален**
   - Согласно документации, `merchantId` - опциональный параметр
   - Можно попробовать без него

## Решение

### Вариант 1: Убрать merchantId из запроса (текущее решение)

Код обновлен - `merchantId` не передается в запрос, так как он опционален.

### Вариант 2: Уточнить правильный merchantId у техподдержки

Нужно спросить:
- "Для API интернет-эквайринга нужен merchantId 15 символов. Вы дали MB0002168266 (13 символов) - это QR-код для СБП. Где найти правильный merchantId для API?"

## Статус

- ⏳ Попробуем без merchantId
- ⏳ Если не сработает - уточним у техподдержки правильный 15-символьный merchantId
